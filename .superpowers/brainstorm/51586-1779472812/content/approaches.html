Three Approaches to Fix Intuitiveness

All 6 tabs feel confusing — here are 3 ways to fix it, from conservative to structural

A

Rename + Explain in Place

Keep 6 tabs, fix the confusion with better names and a one-line context card at the top of each tab that explains what it is and what to do. Collapse dense filter bars behind a "Filters" button.


Rename: Discover → New Releases  ·  Browse → All Videos  ·  Radar → Artist Tracker

Add: One-line tab intro card (dismissible), metric tooltips on Score / Velocity / Engagement

Pros

  • Minimal code change
  • Keeps all current features intact

Cons

  • 6 tabs still feels like a lot
  • Discover vs Browse distinction is still fuzzy even with new names
B

Merge Similar Tabs → 4 Tabs ★ Recommended

Collapse the 6 tabs into 4 by merging the ones that overlap. Each tab now has a clear, distinct purpose. Add a short descriptor line under each tab name in the nav.


🎵 Discover
Curated top picks — replaces old Discover + Browse combined
🎸 Artists
Channels + Last.fm tracking — replaces old Artists + Radar
📈 Trends
Monthly charts + language insights — unchanged
📰 Buzz
Reddit + news — unchanged

Also: Add a subtitle line under each tab label in the nav. Collapse filters behind "Filter" button by default. Add metric tooltips.

Pros

  • Each tab has a clear, distinct job
  • No more "which tab for artists?" confusion
  • Less nav clutter

Cons

  • Moderate rework — Discover tab needs to absorb Browse's full-list view
  • Artists tab gains Last.fm data columns
C

Intent-First Single Feed

Remove tabs entirely. Replace with a persistent intent bar at the top (4 buttons: Find Music / Find Artists / Trends / Buzz). Content below updates based on selection. Deep-dive views (all videos list, artist tracker, charts) accessible via "See all" links within each section.


🎵 Find Music 🎸 Artists 📈 Trends 📰 Buzz
Hero cards + ranked list below, language filter inline. "Browse all 214 videos →" at the bottom.

Pros

  • Most intuitive — no nav decision needed
  • Mirrors the intent picker logic that already exists

Cons

  • Largest rewrite
  • Loses tab memory (active tab persists on refresh)
  • Power users may miss dedicated Browse/Radar views
